# samectx 项目演进历史与上下文

## 项目概述

**samectx** 是一个上下文管理工具，用于在 TRAE、Cursor、Claude Code 等 AI 工具中整理对话上下文并保存到本地目录，支持多用户协作。

**核心理念**：上下文即代码 - 对话上下文应该作为项目的一部分被保存、版本控制和共享。

## 当前版本功能

### 核心功能

| 功能 | 说明 |
|------|------|
| 上下文保存 | 保存对话中的关键任务、关键点、关键决策到本地 |
| 多用户协作 | 每个用户的笔记保存在独立子目录 |
| 用户名识别 | 配置文件 > Git 配置 > 环境变量 > 系统用户名 |
| SKILL 协作 | SKILL 分析对话，npm 保存笔记 |

### 命令列表

```bash
samectx init              # 初始化项目
samectx sync [options]    # 保存上下文
samectx list [options]    # 列出笔记
samectx config [options]  # 配置管理
```

### sync 命令参数

| 参数 | 简写 | 说明 |
|------|------|------|
| `--project <name>` | `-p` | 指定项目名称 |
| `--dir <path>` | `-d` | 指定笔记目录 |
| `--tasks <list>` | `-t` | 关键任务列表（分号分隔） |
| `--keypoints <list>` | `-k` | 关键点列表（分号分隔） |
| `--decisions <list>` | `-D` | 关键决策列表（分号分隔） |
| `--content <text>` | `-c` | 完整对话内容（自动分析） |

## 项目演进历程

### 阶段一：SKILL 方案（最初设计）

**设计思路**：采用 TRAE SKILL 方案，通过静态 `.md` 文件实现上下文同步功能。

**遇到的问题**：SKILL 只支持通过静态 `.md` 文件触发同步行为，**无法通过执行脚本触发同步行为**。

### 阶段二：npm 方案（第一次重构）

**设计思路**：转为开发独立的 npm 包，通过命令行工具实现同步功能。

**遇到的问题**：需要用户每次手动输入命令触发同步行为，无法做到智能效果。

### 阶段三：双方案（Gist 版本）

**设计思路**：结合 SKILL 和 npm 的优势，同步到 GitHub Gist。

**遇到的问题**：
1. Gist 有文件数量限制
2. 需要用户配置 GitHub Token
3. 隐私问题：笔记上传到云端

### 阶段四：本地存储版本（当前状态）

**设计思路**：
- 移除 Gist 同步功能
- 保存到本地目录 `samectx-notes/{username}/`
- 支持多用户协作
- 通过 Git 实现跨设备同步

**优势**：
1. ✅ 无需配置 Token
2. ✅ 笔记作为项目代码的一部分
3. ✅ 支持 Git 版本控制
4. ✅ 多用户协作友好

### 阶段五：SKILL + npm 协作模式（最新）

**设计思路**：SKILL 分析对话提取关键信息，npm 保存结构化笔记。

```
┌─────────────────────────────────────────────────────────────┐
│                    TRAE / Cursor / Claude Code              │
│                                                             │
│  ┌─────────────┐     ┌─────────────────────────────────┐   │
│  │   AI 对话   │ ──→ │ SKILL 分析对话，提取关键信息    │   │
│  │             │     │ - 关键任务 (tasks)              │   │
│  │             │     │ - 关键点 (keyPoints)            │   │
│  │             │     │ - 关键决策 (decisions)          │   │
│  └─────────────┘     └──────────────┬──────────────────┘   │
│                                     │                       │
│                                     ↓                       │
│                     ┌───────────────────────────────────┐   │
│                     │ 生成命令:                         │   │
│                     │ samectx sync --tasks "..."        │   │
│                     │   --keypoints "..." --decisions "..."│
│                     └───────────────────────────────────┘   │
└─────────────────────────────────────────────────────────────┘
                                     │
                                     ↓
┌─────────────────────────────────────────────────────────────┐
│                    npm-samectx (独立进程)                    │
│                                                             │
│  samectx sync --tasks "..." --keypoints "..." --decisions "..."│
│                     │                                       │
│                     ↓                                       │
│  ┌─────────────────────────────────────────────────────┐   │
│  │ 保存到本地文件                                       │   │
│  │ samectx-notes/{username}/context_xxx.json           │   │
│  └─────────────────────────────────────────────────────┘   │
└─────────────────────────────────────────────────────────────┘
```

## 技术决策记录

### 决策 1：为什么移除 Gist 同步？

| 问题 | 说明 |
|------|------|
| 文件限制 | Gist 有文件数量限制 |
| Token 配置 | 需要用户配置 GitHub Token |
| 隐私问题 | 笔记上传到云端 |
| 协作困难 | 多用户协作不友好 |

### 决策 2：为什么保存到本地目录？

| 优势 | 说明 |
|------|------|
| 无需配置 | 不需要 Token |
| 版本控制 | 通过 Git 管理 |
| 项目资产 | 笔记作为项目的一部分 |
| 多用户友好 | 每个用户独立子目录 |

### 决策 3：为什么采用 SKILL + npm 协作？

| 优势 | 说明 |
|------|------|
| 智能提取 | SKILL 分析对话提取关键信息 |
| 结构化保存 | npm 保存结构化笔记 |
| 用户体验 | 用户只需执行生成的命令 |

## 用户名识别优先级

```
1. 配置文件用户名（最高优先级）
   ↓ 未配置
2. Git 配置用户名
   ↓ 未配置
3. 环境变量 SAMECTX_USER
   ↓ 未配置
4. 系统用户名（兜底）
```

## SKILL 调用指南

### SKILL 职责

1. 分析当前对话内容
2. 提取关键任务、关键点、关键决策
3. 生成带参数的 `samectx sync` 命令
4. 提示用户执行命令

### SKILL 生成命令示例

```bash
# 示例 1：完整关键信息
samectx sync --tasks "实现用户登录功能;修复登录页面样式" --keypoints "使用JWT进行认证" --decisions "选择bcrypt加密密码"

# 示例 2：只有任务
samectx sync --tasks "完成API开发;编写单元测试"

# 示例 3：传入完整对话
samectx sync --content "用户要求实现登录功能..."
```

### SKILL 最佳实践

1. **智能提取**：自动识别对话中的关键信息
2. **分号分隔**：多个项目用分号 `;` 分隔
3. **简洁明了**：生成的命令应该易于理解
4. **提示执行**：明确告诉用户执行命令
